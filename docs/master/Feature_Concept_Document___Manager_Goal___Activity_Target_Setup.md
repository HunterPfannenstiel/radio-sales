A sales manager needs a simple way to set clear expectations for each rep so the rep’s daily dashboard can tell them exactly what to do to stay on pace. This feature lets a sales manager create and update each rep’s sales goals and weekly activity targets, and it establishes the accountability baseline for coaching. The manager sets these numbers in consultation with the rep, but the rep cannot change them in the system. The intent is to keep the product seller-focused and execution-focused: the manager defines the standards, the rep drives their own activity, and both parties use the same numbers as the shared truth.

For each rep, the manager can set a monthly sales goal and an annual sales goal. These goals are used elsewhere in the product to calculate percent to goal and to show whether the rep is gaining or losing altitude toward their goal. The manager can change goals at any time, including mid-month. When a goal is changed, the system immediately recalculates percent to goal for the current month based on the current goal (the system reflects the current truth, not a historical audit of prior goal values).

For weekly activity, the manager sets targets per rep for calls per week and asks per week. The system assumes a Monday through Friday work week for pacing and progress calculations. Weekly activity is tracked by week number across the year. The manager can change weekly activity targets at any time, including mid-week. When targets are changed, pacing and progress for the current week immediately recalculates based on the current targets.

The manager’s role for this feature is limited to setting and editing goals and targets for the reps they manage. The rep’s role is view-only: reps can see their current goals and activity targets so they understand what the manager expects and what the dashboard is measuring against, but they cannot edit those values. This feature does not generate notifications when goals or targets change; the product remains check-the-dashboard rather than alert-driven.

This feature assumes each rep belongs to one sales manager at a time for the purposes of goal and target setting. If a rep changes managers, their goals and targets are managed by their current manager going forward (the system behavior is based on the current manager-rep relationship).

Questions:

- How will users create their accounts and get set up?

- One method I have in mind is account claiming. Users will have to enter a unique code that they are given which is linked to their account that has been set up for them in the system. When the user enters their code they can see the information pertaining to the account and “claim” it by setting a username/password. They can also have the ability to change any of the information if it is incorrect.

- Users can then create “claimable accounts” for roles that are beneath them, i.e. a manager can create a “Rep” account for a representative that they manage. The manager can then give the rep the unique code that was generated and once the rep claims the account they will automatically be assigned to the manager.
