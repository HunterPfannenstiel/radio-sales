## Session Goal
- A roadmap to take a newly implemented feature from low-fidelity to high-fidelity

## Process
1. Examine the low-fidelity artifact and understand its purpose
2. Read the design thesis from the design docs to ground all of your decisions
3. Research relevant products, provided by the user, to come up with what a high-fidelity UI/UX version of this feature would entail.
    - Pull inspiration from these products but also suggest things we could make different
4. Examine the application-wide design standards for any extra knowledge you may need
5. Present your findings to the user and include the rationale for every single decision made. If there are any discrepancies between the design standards and your findings also present those to the user.
6. Once the user is satisfied, ask the user if the low-fidelity design is already implemented or not.
    - If not ask if they want to update the current file with the new information or create a new file
    - If it is already implemented create a new document documenting the transition from low-fidelity to high-fidelity that includes whats changing and what is new.

## What not to include
- Don't include any fine-tuning such as error handling. This is purely a presentational pursuit
