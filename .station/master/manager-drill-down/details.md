# Details

## Page Concept

The drill-down answers one question: "Why is this rep behind?"

It displays four diagnostic metrics, each with a goal-relative health signal. The manager scans the health signals to identify which lever is off, then clicks into the underlying data for that metric. The detail view surfaces the inputs to the formula — not a prose list of interactions, but the data that went into the number.

Health is always measured against the rep's own goal, not team average.

## The Four Metrics

### Activity Pace

Whether the rep is generating enough meaningful sales activity relative to where they should be in the month.

> **Needs Clarity** — exact definition of what constitutes a qualifying interaction is unresolved. Working assumption: interactions where the rep has logged a confidence level or a closed sale. A straight-line projection of the monthly target determines expected pace at any point in the month.

### Asks

Count of interactions where the rep logged an ask amount this month, benchmarked against a manager-set weekly ask target.

Managers set a weekly ask target per rep. Health is straightforward: actual ask count vs. that target. No system-wide threshold needed.

### Closing Ratio

Interactions logged with an outcome of "yes" (closed) divided by total asks this month.

The detail view must surface ask volume alongside the ratio — a rep with few asks can have a misleading closing ratio. Both numbers need to be visible together.

### Average Account Value

> **Note: this is actually average ask value**, not average closed deal value. May want to revisit whether closed deal value is more appropriate.

Total ask amounts divided by number of asks this month. Catches reps who are hitting their ask count but sandbagging on deal size.

## Detail Views

Each metric links into a detail view that surfaces the inputs to its formula. The goal is diagnostic clarity, not a full activity log. The manager should be able to see what drove the number without wading through raw entries.

## Dollar Visibility

Managers have visibility into rep-level dollar amounts in drill-down views — sold, projected, and ask amounts. This supports coaching conversations around budget levels, deal size, and realistic pacing.

## Open Questions

- **Activity Pace definition** — what exactly qualifies as a meaningful interaction for pace calculation? Needs alignment with how reps use the logging system in practice.
- **Average Account Value vs. Average Ask Value** — should this metric reflect ask amounts or closed deal amounts? Each tells a different story.
- **Health thresholds** — specific thresholds for each metric's health signal are deferred to a later session.
- **Closing Ratio context** — what is the closing ratio benchmarked against for health? Rep's own historical average, a manager-set target, or a system default?
